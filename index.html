📚 1. KELAS BUKU - CONSTRUCTOR DASAR

🔍 PENJELASAN LENGKAP:

Constructor adalah method khusus yang akan dipanggil secara otomatis ketika sebuah objek dibuat. Tujuan utamanya adalah untuk menginisialisasi nilai awal dari atribut-atribut yang dimiliki oleh kelas tersebut.

Parameterized Constructor adalah constructor yang menerima parameter untuk mengisi nilai awal atribut. Dalam Dart, kita bisa menggunakan this.namaAtribut pada parameter constructor untuk langsung mengisi nilai atribut tanpa perlu menulis kode tambahan di dalam body constructor.

Mengapa Penting? Tanpa constructor, kita harus mengisi nilai atribut satu per satu setelah objek dibuat, yang bisa menyebabkan inkonsistensi data atau lupa mengisi nilai penting. Constructor memastikan bahwa setiap objek lahir dengan keadaan yang valid dan siap digunakan.

Contoh Dunia Nyata: Bayangkan Anda ingin mendata buku di perpustakaan. Setiap buku pasti memiliki judul. Dengan constructor, kita memaksa setiap objek Buku untuk memiliki judul sejak awal, tidak boleh ada buku tanpa judul.

💡 POIN PENTING: Constructor memiliki nama yang sama dengan nama kelas. Tidak memiliki tipe return (bahkan void tidak boleh). Dipanggil dengan keyword new (opsional di Dart).
📁 buku.dart DART 
// Nama   : USMAN YAKIN TRIO SINADA
// NIM    : 251410006

class Buku {
  String judul;

  // Parameterized Constructor
  // 'this.judul' artinya: parameter judul akan langsung
  // diisikan ke field judul milik objek yang sedang dibuat
  Buku(this.judul);
}

void main() {
  // Membuat objek Buku dengan mengisi judul di constructor
  Buku buku1 = Buku("Pemrograman Dart untuk Pemula");
  Buku buku2 = Buku("Clean Code - Robert C. Martin");
  
  // Menampilkan judul buku
  print("Judul Buku 1: ${buku1.judul}");
  print("Judul Buku 2: ${buku2.judul}");
  
  // Output:
  // Judul Buku 1: Pemrograman Dart untuk Pemula
  // Judul Buku 2: Clean Code - Robert C. Martin
}

🚗 2. KELAS MOBIL - NAMED CONSTRUCTOR

🔍 PENJELASAN LENGKAP:

Named Constructor adalah constructor yang memiliki nama khusus selain nama kelas. Ini memungkinkan kita memiliki beberapa cara berbeda untuk membuat objek dari kelas yang sama.

Dalam Dart, named constructor ditulis dengan sintaks: ClassName.constructorName(). Ini sangat berguna ketika kita ingin membuat objek dengan cara yang lebih deskriptif atau ketika kita perlu menginisialisasi objek dari sumber data yang berbeda (seperti dari JSON, dari database, atau dengan nilai default tertentu).

Kapan Menggunakan Named Constructor?

  • Ketika kita ingin memberikan nama yang bermakna pada cara pembuatan objek (misal: Mobil.baru(), Mobil.bekas())
  • Ketika membuat objek dari format data lain (JSON, XML, Map)
  • Ketika kita ingin menyediakan nilai default yang berbeda untuk skenario berbeda
  • Untuk implementasi pola desain seperti Factory atau Singleton

Contoh Dunia Nyata: Di showroom mobil, ada beberapa cara mendapatkan mobil: mobil baru dari pabrik, mobil bekas, atau mobil impor. Named constructor mencerminkan berbagai "sumber" mobil ini.

💡 POIN PENTING: Named constructor tidak bisa dipanggil dengan keyword new jika kita ingin membuat objek const. Named constructor juga bisa bersifat factory atau const.
📁 mobil.dart DART 
class Mobil {
  String? merk;
  int? tahun;

  // Parameterized Constructor (Constructor biasa)
  Mobil(this.merk, this.tahun);

  // Named Constructor: Mobil Baru
  // Digunakan untuk membuat mobil dengan merk dan tahun default
  Mobil.baru() {
    merk = 'Nissan Skyline GT-R R34';
    tahun = 2002;
  }

  // Named Constructor: dari JSON
  // Berguna ketika mengambil data dari API atau file
  Mobil.dariJson(Map<String, dynamic> json) {
    merk = json['merk'];
    tahun = json['tahun'];
  }

  @override
  String toString() {
    return 'Mobil{merk: $merk, tahun: $tahun}';
  }
}

void main() {
  // 1. Menggunakan Parameterized Constructor
  Mobil mobil1 = Mobil('Toyota', 2018);
  print('Mobil 1: $mobil1');

  // 2. Menggunakan Named Constructor baru()
  Mobil mobil2 = Mobil.baru();
  print('Mobil 2: $mobil2');

  // 3. Menggunakan Named Constructor dariJson()
  Map<String, dynamic> dataMobil = {'merk': 'Honda', 'tahun': 2020};
  Mobil mobil3 = Mobil.dariJson(dataMobil);
  print('Mobil 3: $mobil3');
  
  // Output:
  // Mobil 1: Mobil{merk: Toyota, tahun: 2018}
  // Mobil 2: Mobil{merk: Nissan Skyline GT-R R34, tahun: 2002}
  // Mobil 3: Mobil{merk: Honda, tahun: 2020}
}
📤 OUTPUT PROGRAM:
Mobil 1: Mobil{merk: Toyota, tahun: 2018} Mobil 2: Mobil{merk: Nissan Skyline GT-R R34, tahun: 2002} Mobil 3: Mobil{merk: Honda, tahun: 2020}

🎨 3. KELAS WARNA - CONSTANT CONSTRUCTOR

🔍 PENJELASAN LENGKAP:

Constant Constructor adalah constructor yang membuat objek yang bersifat immutable (tidak bisa diubah) dan dioptimasi oleh compiler. Objek yang dibuat dengan constant constructor akan ditempatkan di memori yang sama jika nilainya identik. Ini disebut canonicalization.

Syarat membuat constant constructor:

  • Semua field harus bersifat final (tidak bisa diubah setelah diinisialisasi)
  • Constructor harus diawali keyword const
  • Tidak boleh memiliki body (hanya initializer list)

Mengapa Perlu Constant Constructor?

  • Hemat Memori: Objek dengan nilai yang sama hanya dibuat sekali
  • Thread Safety: Karena tidak bisa diubah, aman digunakan di banyak thread
  • Prediktabilitas: Nilai tidak akan berubah sepanjang program berjalan
  • Optimasi Performa: Compiler bisa melakukan berbagai optimasi

Contoh Dunia Nyata: Warna dalam palet. Warna Merah dengan kode RGB(255,0,0) akan selalu sama di mana pun. Tidak perlu membuat banyak objek untuk warna merah yang sama.

💡 POIN PENTING: Untuk mendapatkan manfaat canonicalization, kita harus membuat objek dengan keyword const, bukan new atau tanpa keyword.
📁 warna.dart DART 
class Warna {
  final int r, g, b;

  // Constant Constructor
  // Semua field harus final
  const Warna(this.r, this.g, this.b);

  @override
  String toString() {
    return 'Warna(r: $r, g: $g, b: $b)';
  }
}

void main() {
  // Membuat objek const
  const warnaMerah = Warna(255, 0, 0);
  const warnaHijau = Warna(0, 255, 0);
  const warnaBiru = Warna(0, 0, 255);

  print('Merah: $warnaMerah');
  print('Hijau: $warnaHijau');
  print('Biru : $warnaBiru');

  // Demonstrasi Canonicalization
  // Objek dengan nilai yang sama akan menggunakan memory yang sama
  const warnaMerah2 = Warna(255, 0, 0);
  
  print('Apakah warnaMerah dan warnaMerah2 sama? 
        ${identical(warnaMerah, warnaMerah2)}'); // true
  
  // Output:
  // Merah: Warna(r: 255, g: 0, b: 0)
  // Hijau: Warna(r: 0, g: 255, b: 0)
  // Biru : Warna(r: 0, g: 0, b: 255)
  // Apakah sama? true
}
📤 OUTPUT PROGRAM:
Merah: Warna(r: 255, g: 0, b: 0) Hijau: Warna(r: 0, g: 255, b: 0) Biru : Warna(r: 0, g: 0, b: 255) Apakah warnaMerah dan warnaMerah2 sama? true

📐 4. KELAS TITIK - INITIALIZER LIST

🔍 PENJELASAN LENGKAP:

Initializer List adalah fitur di Dart yang memungkinkan kita mengeksekusi kode sebelum constructor body dijalankan. Biasanya digunakan untuk:

  • Validasi nilai parameter sebelum digunakan
  • Inisialisasi field final dengan perhitungan
  • Memanggil constructor lain (constructor chaining)
  • Memastikan nilai dalam rentang tertentu

Sintaks initializer list ditulis setelah parameter constructor, dipisahkan dengan titik dua (:), dan sebelum kurung kurawal body. Beberapa inisialisasi dipisahkan dengan koma.

Urutan Eksekusi:

  1. Initializer list dijalankan
  2. Constructor body dijalankan
  3. Objek siap digunakan

Contoh Dunia Nyata: Saat membuat titik koordinat, kita mungkin ingin memastikan semua koordinat bernilai positif (misal untuk sistem yang hanya bekerja di kuadran I). Initializer list bisa mengubah nilai negatif menjadi positif sebelum objek jadi.

💡 POIN PENTING: Initializer list tidak bisa mengakses this karena objek belum selesai dibuat. Tapi bisa mengakses parameter constructor.
📁 titik.dart DART 
class Titik {
  final double x, y;

  // Initializer list untuk memastikan koordinat bernilai absolut
  // Kode setelah titik dua (:) dieksekusi SEBELUM body constructor
  Titik(double x, double y)
      : x = x.abs(),  // Ubah ke positif jika negatif
        y = y.abs() {
    // Body constructor - dieksekusi SETELAH initializer list
    print('✅ Titik (${this.x}, ${this.y}) berhasil dibuat.');
  }

  // Method untuk menghitung jarak dari titik (0,0)
  double jarakDariNol() {
    // Rumus Pythagoras: sqrt(x² + y²)
    return (x * x + y * y).sqrt();
  }
}
