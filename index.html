📚 1. Kelas Buku - Constructor Dasar

🔍 PENJELASAN LENGKAP:

Constructor adalah method spesial yang secara otomatis dipanggil ketika sebuah objek dibuat dari suatu kelas. Tujuan utamanya adalah untuk menginisialisasi nilai awal dari atribut-atribut yang dimiliki oleh kelas tersebut. Dalam pemrograman berorientasi objek, constructor memegang peranan penting karena memastikan bahwa setiap objek yang diciptakan berada dalam keadaan yang valid dan siap digunakan.

Parameterized Constructor adalah constructor yang menerima parameter untuk mengisi nilai awal atribut. Dalam Dart, kita bisa menggunakan sintaks this.namaAtribut pada parameter constructor untuk secara langsung mengisi nilai atribut tanpa perlu menulis kode tambahan di dalam body constructor. Ini dikenal dengan istilah "initializing formal parameters".

Mengapa Constructor Penting? Bayangkan jika kita harus mengisi nilai atribut satu per satu setelah objek dibuat. Ini rentan terhadap kesalahan lupa mengisi atribut penting, atau bahkan inkonsistensi data. Constructor memaksa kita untuk menyediakan data yang diperlukan sejak awal, sehingga objek selalu dalam keadaan lengkap dan valid.

Contoh Dunia Nyata: Saat mendata buku di perpustakaan, setiap buku pasti memiliki judul. Tidak ada buku tanpa judul. Dengan constructor, kita memastikan bahwa setiap objek Buku yang dibuat harus memiliki judul. Ini mencerminkan aturan bisnis "setiap buku harus memiliki judul" langsung ke dalam kode.

📌 POIN PENTING: Constructor memiliki nama yang sama dengan nama kelas. Tidak memiliki tipe return (bahkan void tidak diperbolehkan). Dalam Dart, keyword new bersifat opsional saat memanggil constructor.

💡 ANALOGI SEDERHANA: Constructor seperti "kartu identitas" saat lahir. Setiap bayi yang lahir harus memiliki nama, tanggal lahir, dan orang tua. Constructor memastikan data dasar ini tercatat sebelum bayi (objek) bisa berinteraksi dengan dunia.

📁 buku.dart DART

class Buku {
  String judul;

  // Parameterized Constructor
  Buku(this.judul);
}

void main() {
  // Membuat objek dari kelas Buku
  Buku buku1 = Buku("Pemrograman Dart untuk Pemula");

  print("Judul Buku: ${buku1.judul}");
}

🚗 2. Kelas Mobil - Named Constructor

🔍 PENJELASAN LENGKAP:

Named Constructor adalah constructor yang memiliki nama khusus selain nama kelas. Fitur ini memungkinkan kita memiliki beberapa cara berbeda untuk membuat objek dari kelas yang sama, masing-masing dengan nama yang deskriptif sesuai tujuannya.

Dalam Dart, named constructor ditulis dengan sintaks: ClassName.constructorName(). Ini sangat berguna ketika kita perlu menginisialisasi objek dari berbagai sumber data atau dengan berbagai skenario. Named constructor dapat digunakan untuk:

  • Membuat objek dengan nilai default tertentu
  • Membuat objek dari format data lain seperti JSON atau Map
  • Mengimplementasikan pola desain seperti Factory atau Singleton
  • Memberikan nama yang bermakna pada cara pembuatan objek

Perbedaan dengan Constructor Biasa: Constructor biasa (tanpa nama) hanya memiliki satu cara pembuatan objek. Named constructor memberikan fleksibilitas untuk memiliki multiple constructor dengan perilaku inisialisasi yang berbeda-beda.

Contoh Dunia Nyata: Dalam showroom mobil, ada beberapa cara mendapatkan mobil: mobil baru dari pabrik (dengan spesifikasi default), mobil bekas (dengan riwayat pemakaian), atau mobil impor (dengan dokumen khusus). Named constructor mencerminkan berbagai "jalur" mendapatkan mobil ini.

📌 POIN PENTING: Named constructor tidak bisa dipanggil dengan keyword new jika kita ingin membuat objek const. Named constructor juga bisa bersifat factory atau const, tergantung kebutuhan.

💡 ANALOGI SEDERHANA: Named constructor seperti "resep masakan" yang berbeda untuk hidangan yang sama. Misalnya, nasi goreng bisa dibuat dengan resek spesial, resep hemat, atau resep pedas. Semua menghasilkan nasi goreng, tapi dengan cara yang berbeda.

📁 mobil.dart DART 
class Mobil {
  String? merk;
  int? tahun;

  // Parameterized Constructor
  Mobil(this.merk, this.tahun);

  // Named Constructor: Mobil Baru
  Mobil.baru() {
    merk = 'Nissan Skyline GT-R R34';
    tahun = 2002;
  }

  // Named Constructor: dari JSON
  Mobil.dariJson(Map<String, dynamic> json) {
    merk = json['merk'];
    tahun = json['tahun'];
  }
}

🎨 3. Kelas Warna - Constant Constructor

🔍 PENJELASAN LENGKAP:

Constant Constructor adalah constructor yang membuat objek yang bersifat immutable (tidak dapat diubah setelah dibuat). Objek yang dibuat dengan constant constructor akan ditempatkan di memori yang sama jika nilainya identik. Fenomena ini disebut canonicalization.

Syarat mutlak untuk membuat constant constructor:

  • Semua field dalam kelas harus bersifat final (nilai tidak bisa diubah setelah diinisialisasi)
  • Constructor harus diawali dengan keyword const
  • Tidak boleh memiliki body constructor (hanya initializer list yang diperbolehkan)

Mengapa Perlu Constant Constructor?

  • Hemat Memori: Objek dengan nilai yang sama hanya dibuat sekali, menghemat penggunaan memori
  • Thread Safety: Karena immutable, objek aman digunakan di lingkungan multi-threading
  • Prediktabilitas: Nilai objek tidak akan berubah sepanjang program berjalan
  • Optimasi Performa: Compiler dapat melakukan berbagai optimasi pada objek const

Contoh Dunia Nyata: Warna dalam palet. Warna Merah dengan kode RGB(255,0,0) akan selalu sama di mana pun. Tidak perlu membuat banyak objek untuk warna merah yang sama. Dengan constant constructor, kita cukup membuat satu objek untuk warna merah dan menggunakannya di banyak tempat.

📌 POIN PENTING: Untuk mendapatkan manfaat canonicalization, kita harus membuat objek dengan keyword const, bukan new atau tanpa keyword. Jika dibuat tanpa const, objek baru akan tetap dibuat meskipun nilainya sama.

💡 ANALOGI SEDERHANA: Constant constructor seperti "nomor induk" yang unik. Meskipun ada banyak KTP dengan nama berbeda, setiap nomor induuk hanya satu. Objek const bekerja seperti itu - nilai yang sama menghasilkan "nomor induk" memori yang sama.

📁 warna.dart DART 
class Warna {
  final int r, g, b;

  // Constant Constructor
  const Warna(this.r, this.g, this.b);

  @override
  String toString() {
    return 'Warna(r: $r, g: $g, b: $b)';
  }
}

void main() {
  const warnaMerah = Warna(255, 0, 0);
  const warnaHijau = Warna(0, 255, 0);
  print(warnaMerah);
}

📐 4. Kelas Titik - Initializer List

🔍 PENJELASAN LENGKAP:

Initializer List adalah fitur di Dart yang memungkinkan kita mengeksekusi kode sebelum constructor body dijalankan. Ini adalah tahap inisialisasi yang terjadi segera setelah parameter constructor diterima, tetapi sebelum kode di dalam kurung kurawal {} dieksekusi.

Initializer list sangat berguna untuk:

  • Validasi parameter - Memeriksa apakah nilai parameter valid sebelum digunakan
  • Manipulasi nilai awal - Mengubah nilai parameter sebelum diassign ke field
  • Inisialisasi field final - Field final harus diinisialisasi sebelum constructor body, initializer list adalah tempat yang tepat
  • Memanggil constructor lain - Constructor chaining dengan this() atau super()
  • Assertions - Memastikan kondisi tertentu terpenuhi

Urutan Eksekusi Constructor:

  1. Initializer list dijalankan (dipisahkan dengan titik dua : setelah parameter)
  2. Constructor body dijalankan (kode di dalam {})
  3. Objek siap digunakan

Contoh Dunia Nyata: Saat membuat titik koordinat untuk sistem yang hanya bekerja di kuadran I (x dan y positif), initializer list bisa mengubah nilai negatif menjadi positif sebelum objek jadi. Ini seperti "penjaga pintu" yang memastikan data yang masuk sesuai aturan sebelum diizinkan masuk.

📌 POIN PENTING: Initializer list tidak bisa mengakses this karena objek belum selesai dibuat. Namun, kita bisa mengakses parameter constructor. Initializer list dieksekusi dari kiri ke kanan.

💡 ANALOGI SEDERHANA: Initializer list seperti "pos pemeriksaan" sebelum memasuki gedung. Semua orang diperiksa dan disaring dulu di pintu masuk (initializer list), baru setelah itu mereka bisa masuk dan beraktivitas di dalam gedung (constructor body).

📁 titik.dart DART 
class Titik {
  final double x, y;

  // Initializer list untuk memastikan koordinat bernilai absolut
  Titik(double x, double y)
      : x = x.abs(),
        y = y.abs() {
    print('Titik (${this.x}, ${this.y}) berhasil dibuat.');
  }
}

👨‍🎓 5. Kelas Mahasiswa - Named Constructor

🔍 PENJELASAN LENGKAP:

Kelas Mahasiswa ini mendemonstrasikan penggunaan named constructor dengan berbagai sumber data. Named constructor memberikan fleksibilitas dalam menciptakan objek dari format data yang berbeda-beda.

Tiga Cara Membuat Objek Mahasiswa:

  • Default Constructor: Cara standar dengan mengisi semua parameter secara langsung. Ini adalah constructor utama yang paling sering digunakan.
  • Constructor dariMap(): Berguna saat mengambil data dari database, API, atau file konfigurasi yang biasanya berbentuk Map (key-value pairs).
  • Constructor tanpaNama(): Untuk situasi di mana privasi diperlukan, misalnya saat menampilkan data di forum publik tanpa menampilkan nama asli.

Keuntungan Named Constructor: Kode menjadi lebih ekspresif dan mudah dipahami. Ketika membaca Mahasiswa.dariMap(data), kita langsung tahu bahwa objek dibuat dari data Map. Dibandingkan jika hanya menggunakan satu constructor, kita harus menebak-nebak dari parameter yang diberikan.

Contoh Dunia Nyata: Dalam sistem akademik, data mahasiswa bisa datang dari berbagai sumber: form pendaftaran (input langsung), database (Map), atau untuk keperluan statistik tanpa menampilkan identitas (anonim). Named constructor mengakomodasi semua skenario ini dengan rapi.

📌 POIN PENTING: Named constructor dapat memanggil constructor lain dalam kelas yang sama menggunakan this(). Ini disebut "redirecting constructor" dan membantu menghindari duplikasi kode inisialisasi.

💡 ANALOGI SEDERHANA: Named constructor seperti "jalur khusus" di bandara. Ada jalur untuk penumpang biasa (default), jalur diplomatik (dariMap), dan jalur VIP (tanpaNama). Semuanya menuju ke ruang keberangkatan yang sama, tapi dengan proses yang berbeda.

📁 mahasiswa.dart DART 
class Mahasiswa {
  String nama;
  String nim;
  String jurusan;
  String kelas;

  // Default Constructor
  Mahasiswa(this.nama, this.nim, this.jurusan, this.kelas);

  // Named Constructor: dari Map
  Mahasiswa.dariMap(Map<String, String> data)
      : nama = data['nama']!,
        nim = data['nim']!,
        jurusan = data['jurusan']!,
        kelas = data['kelas']!;

  // Named Constructor: Tanpa Nama
  Mahasiswa.tanpaNama(this.nim, this.jurusan, this.kelas) : nama = '*** ANONIM ***';
}

👤 6. Kelas Person - Enkapsulasi Dasar

🔍 PENJELASAN LENGKAP:

Enkapsulasi adalah salah satu pilar utama dalam Object-Oriented Programming (OOP). Konsep ini berarti menyembunyikan data internal suatu objek dari akses langsung dari luar, dan hanya menyediakan akses melalui method-method tertentu (getter dan setter).

Dalam Dart, enkapsulasi diimplementasikan dengan:

  • Field private: Ditandai dengan underscore _ di depan nama variabel. Field ini hanya bisa diakses dari dalam kelas itu sendiri.
  • Getter: Method yang mengembalikan nilai field private. Biasanya dinamai sama dengan field tanpa underscore.
  • Setter: Method yang mengubah nilai field private, biasanya dilengkapi dengan validasi.

Mengapa Enkapsulasi Penting?

  • Keamanan Data: Mencegah data diubah secara sembarangan oleh kode luar
  • Validasi: Setter dapat memvalidasi nilai sebelum mengubah data
  • Fleksibilitas: Implementasi internal dapat diubah tanpa mempengaruhi kode yang menggunakan kelas
  • Kontrol Akses: Kita bisa menentukan data mana yang boleh dibaca, ditulis, atau keduanya

Contoh Dunia Nyata: Enkapsulasi seperti "ATM". Anda bisa melihat saldo (getter) dan mengambil uang (method), tapi Anda tidak bisa langsung mengubah angka saldo di mesin ATM. Semua akses harus melalui prosedur yang sudah ditentukan.

📌 POIN PENTING: Field dengan underscore _ bersifat private di tingkat library, bukan hanya di tingkat kelas. Artinya, field tersebut tidak bisa diakses dari file Dart lain.

💡 ANALOGI SEDERHANA: Enkapsulasi seperti "koper dengan kunci". Isi koper (data) tidak bisa langsung diambil sembarang orang. Harus melalui pemilik yang membuka kunci (getter/setter) dengan prosedur yang benar.

📁 person.dart DART 
class Person {
  String _nama; // Private field
  int _umur;    // Private field

  Person(this._nama, this._umur);

  // Getter
  String get nama => _nama;
  int get umur => _umur;

  // Setter dengan validasi
  set nama(String namaBaru) {
    if (namaBaru.isNotEmpty) _nama = namaBaru;
  }

  set umur(int umurBaru) {
    if (umurBaru > 0 && umurBaru < 150) _umur = umurBaru;
  }
}

📦 7. Kelas Produk - Validasi pada Setter

🔍 PENJELASAN LENGKAP:

Kelas Produk ini menunjukkan penerapan validasi data melalui setter. Validasi adalah proses memeriksa apakah data yang akan dimasukkan memenuhi aturan atau kriteria tertentu sebelum data tersebut benar-benar disimpan.

Mengapa Validasi Penting? Dalam aplikasi nyata, data yang tidak valid dapat menyebabkan:

  • Kerusakan logika bisnis (misal: harga negatif)
  • Error saat pemrosesan data
  • Inkonsistensi database
  • Masalah keamanan (injection, overflow, dll)

Aturan Validasi dalam Kelas Produk:

  • Nama produk: Minimal 3 karakter. Ini mencegah nama produk yang terlalu pendek seperti "PC" atau "TV" yang bisa membingungkan.
  • Harga produk: Harus lebih besar dari 0. Tidak masuk akal jika ada produk dengan harga nol atau negatif (kecuali dalam skenario tertentu seperti diskon, tapi itu perlu penanganan khusus).

Tempat Validasi: Validasi sebaiknya dilakukan di setter karena:

  • Setter adalah satu-satunya jalan untuk mengubah data (jika field private)
  • Validasi terpusat, tidak perlu diulang di banyak tempat
  • Kelas menjaga integritas datanya sendiri

Contoh Dunia Nyata: Saat mengisi formulir online, ada validasi seperti "nama minimal 3 karakter" atau "harga harus angka positif". Ini melindungi sistem dari data yang tidak masuk akal dan memberikan pengalaman pengguna yang lebih baik.

📌 POIN PENTING: Setter tidak harus selalu menerima perubahan. Jika data tidak valid, setter bisa menolak perubahan (dengan tidak mengubah data) dan memberikan pesan error atau melempar exception.

💡 ANALOGI SEDERHANA: Validasi di setter seperti "satpam" di pintu masuk gedung. Satpam akan memeriksa identitas setiap orang yang mau masuk. Jika identitas tidak sesuai (KTP palsu), satpam akan menolak masuk. Hanya orang dengan identitas valid yang diizinkan.

📁 produk.dart DART 
class Produk {
  String _nama;
  double _harga;

  Produk(this._nama, this._harga);

  set nama(String value) {
    if (value.length >= 3) _nama = value;
  }

  set harga(double value) {
    if (value > 0) _harga = value;
  }
}

🔐 8. Kelas User - Sistem Login

🔍 PENJELASAN LENGKAP:

Kelas User mengimplementasikan sistem autentikasi sederhana dengan enkapsulasi data sensitif (password). Ini adalah contoh penerapan enkapsulasi dalam konteks keamanan aplikasi.

Mengapa Password Harus Dienkapsulasi? Password adalah data yang sangat sensitif. Dalam aplikasi nyata:

  • Password tidak boleh bisa dibaca langsung oleh kode lain (tidak ada getter untuk password)
  • Perubahan password harus melalui prosedur yang aman (verifikasi password lama)
  • Password harus memenuhi kriteria keamanan tertentu (minimal panjang, kombinasi karakter, dll)

Fitur Keamanan dalam Kelas User:

  • Field private: _username, _password, _email tidak bisa diakses langsung dari luar
  • Getter terbatas: Hanya username dan email yang bisa dibaca. Password tidak memiliki getter.
  • Method login: Memverifikasi kredensial tanpa mengekspos password
  • Method gantiPassword: Memerlukan password lama untuk verifikasi, dan memvalidasi password baru (minimal 6 karakter)

Prinsip Keamanan yang Diterapkan:

  • Need to know: Hanya informasi yang diperlukan yang diekspos
  • Defense in depth: Validasi berlapis (password lama + syarat password baru)
  • Least privilege: Method hanya melakukan satu tugas spesifik

Contoh Dunia Nyata: Saat Anda mengganti password di media sosial, Anda diminta memasukkan password lama dulu, lalu password baru minimal 8 karakter dengan kombinasi huruf dan angka. Ini persis seperti yang kita implementasikan di kelas User.

📌 POIN PENTING: Dalam aplikasi produksi, password tidak boleh disimpan dalam bentuk plain text seperti contoh ini. Harus di-hash menggunakan algoritma seperti bcrypt, argon2, atau SHA dengan salt.

💡 ANALOGI SEDERHANA: Kelas User seperti "brankas bank". Anda bisa melihat nomor rekening (username) dan saldo (email), tapi tidak bisa melihat kode brankas (password). Untuk mengganti kode, Anda harus melalui prosedur verifikasi yang ketat.

📁 user.dart DART 
class User {
  String _username;
  String _password;
  String _email;

  User(this._username, this._password, this._email);

  bool login(String username, String password) {
    return username == _username && password == _password;
  }

  void gantiPassword(String lama, String baru) {
    if (lama == _password && baru.length >= 6) {
      _password = baru;
    }
  }
}

🏦 9. Kelas RekeningBank - Simulasi Transaksi

🔍 PENJELASAN LENGKAP:

Kelas RekeningBank adalah simulasi sistem perbankan sederhana yang menerapkan enkapsulasi pada data keuangan yang sensitif (saldo). Ini menunjukkan bagaimana enkapsulasi melindungi integritas data dalam skenario transaksi keuangan.

Mengapa Saldo Harus Dienkapsulasi? Saldo rekening adalah data kritis yang tidak boleh sembarangan diubah. Dalam aplikasi perbankan nyata:

  • Tidak boleh ada kode yang bisa langsung menulis rekening.saldo = 1000000 tanpa melalui prosedur yang benar
  • Setiap perubahan saldo harus tercatat (audit trail)
  • Harus ada validasi untuk mencegah transaksi ilegal (saldo negatif, melebihi limit, dll)

Fitur Transaksi yang Diimplementasikan:

  • deposit(): Menambah saldo. Hanya bisa dilakukan dengan jumlah positif.
  • tarik(): Mengurangi saldo. Memerlukan saldo cukup dan jumlah positif.
  • transfer(): Memindahkan dana ke rekening lain. Melibatkan validasi saldo pengirim dan memodifikasi dua rekening sekaligus.

Validasi dalam Transaksi:

  • Jumlah positif: Tidak ada transaksi dengan jumlah nol atau negatif
  • Saldo cukup: Untuk tarik dan transfer, saldo harus >= jumlah yang diminta
  • Integritas transaksi: Dalam transfer, kedua rekening harus konsisten (pengirim berkurang, penerima bertambah)

Contoh Dunia Nyata: Saat Anda transfer uang melalui mobile banking, Anda tidak bisa langsung mengubah saldo. Anda harus melalui menu transfer, memasukkan nomor rekening tujuan, jumlah, dan PIN. Semua ini adalah bentuk enkapsulasi yang melindungi data keuangan Anda.

📌 POIN PENTING: Dalam aplikasi perbankan nyata, method transfer harus bersifat atomik - jika salah satu operasi gagal (misal: rekening tujuan error), maka seluruh transaksi harus dibatalkan (rollback). Ini biasanya diimplementasikan dengan database transaction.

💡 ANALOGI SEDERHANA: Kelas RekeningBank seperti "ATM". Anda bisa menyetor (deposit), mengambil (tarik), dan transfer uang. Tapi Anda tidak bisa membuka mesin ATM dan mengambil uang seenaknya. Semua transaksi harus melalui prosedur yang sudah ditentukan.

📁 rekening_bank.dart DART 
class RekeningBank {
  String _nomorRekening;
  double _saldo;

  RekeningBank(this._nomorRekening, this._saldo);

  void deposit(double jumlah) {
    if (jumlah > 0) _saldo += jumlah;
  }

  void tarik(double jumlah) {
    if (jumlah > 0 && _saldo >= jumlah) _saldo -= jumlah;
  }

  void transfer(RekeningBank tujuan, double jumlah) {
    if (jumlah > 0 && _saldo >= jumlah) {
      _saldo -= jumlah;
      tujuan._saldo += jumlah;
    }
  }
}
